## Что это

Готовый комплект памяти для Claude Code, собранный по паттерну Karpathy LLM Wiki. Один скрипт ставит хуки `SessionStart` / `PreCompact` / `SessionEnd`, разворачивает Obsidian-vault со схемой каталогов и регистрирует 14 кастомных скиллов: `wiki-ingest`, `wiki-query`, `wiki-lint`, `cross-linker` и другие.

## Установка одной командой

* macOS или Linux, Claude Code Pro.
* Obsidian уже установлен (или будет установлен скриптом, если ты на macOS).

```bash
curl -fsSL https://raw.githubusercontent.com/sergeyramas/karpathy-wiki-bundle/main/install.sh | bash
```

> 💬 **Что произойдёт**
>
> ```text
> 1. ~/.claude-memory-compiler/  — хуки и compiler
> 2. ~/Documents/Jarvis/         — vault со скелетом каталогов
> 3. ~/.claude/skills/           — 14 wiki-скиллов
> 4. ~/.claude/settings.json     — регистрация хуков
> ```

**Зачем:** повторяемый старт. Перенесёшь на новый Mac за 30 секунд тем же curl-ом.

## Первый ingest

После установки в Claude Code пишешь:

```text
/wiki-ingest ~/Downloads/article.pdf
```

Skill распарсит PDF, выделит ключевые сущности и факты, создаст страницу в `wiki/sources/`, обновит сервис-каталог, поправит cross-links. Без копипасты, без ручного формата.

## Запрос знаний

```text
/wiki-query что я знаю про Hetzner?
```

Skill пробежит по всем страницам, соберёт ответ с цитатами и пруфами, при необходимости создаст synthesis-страницу для повторного использования. Это и есть «второй мозг» по Карпати.

## Гигиена и дисциплина

* `wiki-lint` — раз в неделю, ловит противоречия и битые wikilink.
* `cross-linker` — после крупного ingest, пересобирает граф.
* `wiki-status` — что свежего, что устарело.

> 💬 **Идея, которую легко упустить**
>
> ```text
> Главная ценность бандла — не сами скиллы, а то, что они согласованы
> между собой. wiki-ingest пишет в формате, который понимает wiki-query.
> wiki-lint проверяет инварианты. cross-linker дополняет. Без этого
> любой knowledge-base через полгода превращается в свалку.
> ```
