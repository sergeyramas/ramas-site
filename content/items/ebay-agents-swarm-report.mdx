## Контекст

Отчёт по моему участию в Anthropic MacBook Contest. Задача была — показать, как живой бизнес работает поверх Claude-агентов. У меня — два магазина на eBay в США, и они уже больше полугода крутятся без ручной работы. Шаги ниже — путь от первого скрипта до swarm-архитектуры.

## Ручной eBay

Старт — обычный reseller. Купил у поставщика, перепродал на eBay. Узкое горлышко — отправка трекинга и ответы покупателям. На это уходило 2–3 часа в день.

## Первый агент — Closing-бот

* Слушает webhook от eBay о новой оплате
* Проверяет адрес, флаги fraud
* Закрывает заказ статусом «paid, ready to ship»

> 💬 **Урок №1**
>
> ```text
> Один агент = одна функция. Closing-бот ничего не знает про закупку.
> Это отдельная программа. Иначе одна ошибка ложит всю цепочку.
> ```

**Эффект:** -1 час в день.

## Покупатель — Purchasing-бот

После закрытия eBay-заказа Purchasing получает SKU и адрес покупателя, идёт к нужному поставщику (несколько источников в каталоге, для каждого — свой адаптер), проходит чекаут с подменой shipping address. Полностью автономно для известных поставщиков.

## Tracking sync + Buyer messages

* Поставщик пишет «отправил, трек XXX» — агент распарсит письмо
* Прокинет трек в eBay через Trading API
* Покупателю — короткое сообщение «спасибо, ждите»

К концу второго месяца ручной работы в день — 0 минут. Все часы — на принятие решений и расширение каталога.

## Swarm-слой

Дальше встал вопрос: как добавлять новых поставщиков и категории, не переписывая весь стек. Появился swarm:

* **Verifier** проверяет работу всех остальных агентов
* **Researcher** ищет новые SKU по паттерну прибыльности
* **Морфиус** (так его назвал) — оркестратор, разрешает споры между агентами

> 💬 **Урок №2 — самый недооценённый**
>
> ```text
> Без отдельного Verifier агентский pipeline через 2 месяца
> начинает принимать ошибочные решения, которые ты не замечаешь.
> Verifier не делает — он только проверяет, и сам отчитывается
> владельцу. Это спас от двух потерь больше $5k каждая.
> ```

## Безопасность и предохранители

* Все денежные транзакции > $200 — ручное подтверждение в Telegram
* Лимит на категорию в день — мягкий стоп
* Watchdog: если 3 заказа подряд завалились — pause всю purchase-цепочку
