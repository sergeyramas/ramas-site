> **TL;DR.** Стэк, на котором живёт reseller-бизнес из США. Два аккаунта eBay (Georgia primary + Peru secondary) с разной юрисдикцией для distribution риска. Пять автономных ботов закрывают сделки, закупают у поставщиков, синхронят треки, мониторят цены и общаются с покупателями. Человек заходит в систему ~15-20 минут в день — только чтобы добавить новые SKU и проверить алерты в Telegram. **Систему можно поставить под ваш бизнес.** [→ Связаться в Telegram](https://t.me/Sergeyramas)

## Зачем это нужно

Классическая боль reseller'а на eBay:

- Заказ оплачен → надо вручную закрыть сделку, проставить статус, написать покупателю
- Поставщик отгрузил → надо вручную взять трек и вбить в eBay
- Цена у поставщика прыгнула → твой листинг становится убыточным, а ты узнаёшь спустя неделю
- Товар закончился → продал то, чего нет → strike по аккаунту → ban
- 20 одинаковых писем покупателям про tracking и refunds → заняты вечера

При **20-50 заказах в день** всё это превращается в полноценную работу на 6-8 часов. У одного человека — до 30 заказов потолок без выгорания. Стэк убирает эту ручную работу полностью и масштабирует **в 5-10 раз** на одного оператора.

## Что входит в стэк

Пять автономных ботов плюс контрольная панель в Google Sheets и Telegram.

### 1. Closing-автомат

Слушает eBay через Trading API. Как только покупатель оплатил — заказ автоматически:
- Помечается как Paid
- Триггерит Purchasing-bot (см. ниже)
- Записывает SKU + параметры заказа в Google Sheets
- Шлёт алерт в Telegram (для оператора, на случай если что-то пошло не так)

Время от платежа до закупки у поставщика — **под 60 секунд** в среднем.

### 2. Purchasing bot

Самая нетривиальная часть. По SKU из заказа знает, у какого поставщика брать товар (карта `SKU_CATALOG` в Google Sheets — про неё ниже). Бот:
- Открывает сайт поставщика (DirectHomeMedical, CPAPXchange, ResmedDirect, Shopify-магазины)
- Логинится сохранёнными credentials
- Кладёт товар в корзину
- **Подменяет shipping-адрес на адрес покупателя eBay** (ключевая dropshipping-механика)
- Оплачивает корпоративной картой
- Сохраняет supplier order ID в Sheets

Для каждого поставщика — свой адаптер (текущих — 4, добавление нового занимает ~2 часа). Полная автономность: ни одного клика человеком в стандартном случае.

### 3. Tracking sync

Опрашивает поставщиков по supplier order ID. Как только трек появился:
- Прокидывает его в eBay через Trading API → статус заказа обновляется
- Автоматически отправляет покупателю message с tracking
- Помечает в Google Sheets дату отгрузки и ожидаемую дату доставки

Покупатели получают информацию **раньше**, чем у обычных reseller'ов — за это растут seller metrics и Top Rated статус.

### 4. Price/stock monitor

Каждые N часов сверяет каждый твой листинг с источником у поставщика:
- Цена выросла → пересчитывает с учётом маржи и обновляет на eBay
- Цена упала → опционально снижает у тебя (более конкурентный листинг)
- Out of stock → автоматически скрывает листинг (`hide` вместо `delete`, чтобы сохранить sales history)
- Вернулся в наличие → автоматически снимает hide

Защищает от стрика «продал то, чего нет» — главной угрозы reseller'у на eBay.

### 5. Buyer messaging via Trading API

Сообщения покупателю шлются программно — без захода в eBay UI. Отдельные сценарии:
- **Apology** — извинение при задержке
- **Refund explanation** — объяснение возврата с причиной
- **Tracking** — отправка трек-номера
- **Follow-up** — после доставки (просьба о feedback'е)
- **Custom** — на любую input-фразу (используется AI-агент с контекстом ордера)

Все сообщения логируются в Google Sheets для compliance и audit'а.

```
        ┌───────▼─────────┐    ┌──────────────────┐

        └─────────────────┘    └──────────────────┘
                │                       │
                └────────┬──────────────┘
                         ▼
                  ┌──────────────┐
                  │  Telegram    │
                  │  notifier    │
                  └──────────────┘
```

**Стек:**
- **Python 3.11** + asyncio для всех ботов
- **VPS** (Hetzner / DigitalOcean, ~$10/мес) — все процессы крутятся одним systemd-юнитом
- **Google Sheets API** — каталог SKU↔источник, статусы заказов, audit log
- **eBay Trading API** + Inventory API
- **Playwright** — для поставщиков без API (Shopify-магазины, DirectHomeMedical-чат)
- **Telegram Bot API** — алерты в реальном времени
- **Redis** — координация между ботами, дедупликация событий
- **Postgres** — persistent state, retry queue для падающих закупок

Вся секретная конфигурация — в `.env` на VPS, **никаких credentials в коде**, репозиторий приватный.

## Полуавтомат: что делает робот, что человек

Стэк сознательно **полу-автоматический**, не **полный** — потому что 100% автомат на маркетплейсах = бан-риск. Граница между "робот" и "человек" вынесена осознанно.

| Действие | Робот | Человек |
|---|:---:|:---:|
| Закрытие оплаченных заказов | ✅ | |
| Закупка у поставщика | ✅ | |
| Подмена shipping address | ✅ | |
| Tracking sync в eBay | ✅ | |
| Стандартные buyer messages | ✅ | |
| Мониторинг цен и stock | ✅ | |
| Скрытие OOS-листингов | ✅ | |
| **Добавление новых SKU** | | ✅ |
| **Решения по spike-ценам** (если supplier поднял на 30%+) | алерт | принимает решение |
| **Сложные buyer-кейсы** (defect, return, dispute) | drafts ответ | проверяет и отправляет |
| **Договоры с поставщиками** | | ✅ |
| **Стратегия каталога** (что добавить в линейку) | | ✅ |

В среднем оператор тратит **15-20 минут в день** на задачи из правой колонки. Остальное — алерты в Telegram, на которые надо просто отреагировать тапом.

## Кому это подходит

- **Действующим reseller'ам на eBay**, которые уперлись в потолок ручной работы (15-30 заказов/день — выгорание)
- **Тем, кто хочет начать reseller-бизнес правильно** — сразу с инфраструктуры, а не учить руки на каждом заказе
- **Командам с несколькими аккаунтами на разных юрисдикциях** — стэк масштабируется на 2-N аккаунтов одной установкой
- **Бизнесам, у которых есть прямые контракты с поставщиками** — мы интегрируем под ваш список (стандартно 4 адаптера, добавление нового — ~2 часа работы)

**Не подходит:** если ты продаёшь свой собственный товар (не reseller). Тогда не нужен Purchasing bot и логика подмены адреса — там другая архитектура.
