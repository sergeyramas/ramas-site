## Что это

Короткий протокол работы с Claude Code по Борису Чёрному и Андрею Карпатому: важны не только сами команды, а их порядок. Те же шесть фраз, поданные хаотично, сжигают лимит и дают переделки. В правильной последовательности сначала появляется рамка работы, потом проверка, потом код.

## Порядок

1. **«Запускай субагентов»** — для больших задач дробить работу на параллельные сессии со своей зоной ответственности.
2. **«Напиши implementation spec»** — до сборки выбрать один путь реализации и убрать догадки.
3. **«Проинтервьюируй меня»** — закрыть пропущенные решения вопросами и вернуть ответы обратно в spec.
4. **«Проверяй до того, как строить»** — заранее описать, чем и где проверять результат, включая human-in-the-loop для дорогих ошибок.
5. **«Сделай из этого скилл»** — упаковать повторяющийся удачный диалог в SKILL.md, scripts, examples и GOTCHAS.
6. **«Автоматизируй это»** — включать только после фильтров: нужен ли вкус и устроит ли 80% результат. Критичное оставлять под человеком.

## Три слоя

Фразы 1-4 — фундамент: скорость, предсказуемость, спецификация и проверка до первой строки кода. Фраза 5 — самоулучшение: один раз руками исправил процесс, дальше ошибка уходит в скилл. Фраза 6 — мощь плюс риск: автоматизация дает рычаг, но кривой автомат ухудшает рабочий процесс.

## A/B-тест

Тест был на одной задаче: Telegram-бот собирает заявки клиентов и заносит их в Excel. Без скилла Claude Code сразу ушел в код, не задал вопросов и собрал примитивный вариант. Со Spec Kit скиллом порядок стал другим: интервью, спека, подтверждение, субагенты, guard-проверка. Итог — рабочий бот за один промт и без ручной правки кода.
