## О чём это

Первый урок из мульти-агентного интенсива. Цель — за вечер собрать собственного AI-секретаря: бот в Telegram, который читает почту, ставит задачи в календарь и сам пишет ответы черновиками. Без n8n, без low-code — на чистом Python и Claude Code.

## Установка окружения

* macOS или Linux. На Windows — через WSL.
* `python 3.11+`, `node 20+`, аккаунт Claude Code Pro.
* GitHub-аккаунт для приватного репозитория.

> 💬 **Промпт для старта**
>
> ```text
> Я хочу собрать персонального AI-секретаря, который читает мою почту,
> понимает контекст, ставит задачи в календарь и черновики ответов.
> Стек: Python + Claude Agent SDK + Telegram Bot API + Google APIs.
> Создай скелет проекта.
> ```

**Зачем:** дать Claude Code чёткий брифинг до первого `Edit`. Иначе он начнёт «улучшать архитектуру» и закопается на полчаса.

## Каркас агента

* Файл `agent.py` — точка входа.
* `tools/` — модули `gmail`, `calendar`, `telegram`.
* `state/` — JSON-файлы для памяти и контекста.
* `.env` — секреты, в `.gitignore` сразу.

Дальше Claude Code получает промпт «реализуй чтение последних 20 непрочитанных писем Gmail и положи в `state/inbox.json`». 5–7 итераций — модуль работает.

## Связка с Telegram

Бот подключается к группе или личке. Команды:

* `/today` — план на день из календаря
* `/inbox` — топ-3 важных письма с резюме
* `/draft <id>` — черновик ответа

> 💬 **Промпт на интеграцию**
>
> ```text
> Добавь Telegram-бота. Команды /today, /inbox, /draft.
> /draft вызывает Claude через Anthropic API, передаёт письмо
> и возвращает текст ответа в стиле моих прошлых писем.
> ```

**Зачем:** граница — что делает локальный агент, а что — Claude по API. Локальный — IO. Claude — текст и решения.

## Деплой и привычка

* `pm2` или systemd на VPS (Hetzner CX22 за €5/мес хватает).
* Cron: каждые 15 минут пройти inbox + календарь.
* Уведомление в Telegram только если **что-то новое и важное**.

Через неделю ты отвечаешь на 80% писем за пять минут утром, а не за час в течение дня. Дальше — следующий урок: подключаем второго агента, который проверяет работу первого.

## Что внутри полного гайда

Все 24 шага с командами, скрипты, готовые промпты, способы переучить под свой почтовый стиль и схему интеграции с Google Workspace. Хостится на GitHub Pages, открывается в браузере одной ссылкой.
