Gap-анализ: BetaLine v2 — «решения для бизнеса», не «чатик»

2026-07-09. База: 22 компании (раунд 1, блочные паттерны) + 8 каталогов решений + 4 диагностических оффера + дословные боли (03-reviews/pains-verbatim.md).

1. Насыщение (3+ конкурентов — упомянуть, но НЕ лидировать этим)

  • «Интеграция с CRM/1С/мессенджерами» — есть у ВСЕХ (Digital-Pro, NIKTA, OSMI, Code9, Webseed, Automacon…). Подтверждение интуиции Сергея: «интеграция с CRM» как оффер выжжена. Упоминать компактной полосой, не строить на этом блок.
  • «Бот отвечает 24/7» — все. Табличный стейк.
  • «Бесплатный аудит» как фраза-CTA — veonix, it-implant, openii, maksoft, wiseadvice, neuro-bots. Слова выжжены; формат «форма→созвон» выжжен.
  • Каталог по отделам/отраслям — veonix (отделы), generation-ai (13 отраслей), digital-pro (типы агентов).
  • Этапы внедрения с длительностями — digital-pro, OSMI, Code9, mass-ai, Bland.

2. Разрывы (НИКТО не делает — золото дифференциации)

  • G1. Каталог решений по ЗАТЫКАМ-ситуациям. Рынок структурирует по отделам/отраслям/технологиям. Никто — по узнаваемой ситуации клиента («три обращения в трёх каналах», «КП делается 3 часа», «заявка в субботу 21:47»). Дословные боли есть в 03-reviews — язык готов.
  • G2. Диагностика = живое демо продукта. Единственный интерактивный аудитор (scriptogenerator) — опросник на сайте, оторванный от продукта. Никто не отдаёт диагностику в свой реальный работающий бот, где сам аудит доказывает качество продукта и оставляет лида «привязанным» к боту (постоянный канал возврата).
  • G3. Позиционирование «против конструктора». Отзывы на SaleBot/BotHelp полны болей DIY («нет множества необходимых функций», «заумно», «сами починить не можете») — ни один интегратор явно не таргетирует аудиторию «пробовали конструктор — упёрлись». Это точный вход в «кастом».

3. Слабые места (1-2 делают поверхностно — можно сделать лучше)

  • Сквозной процесс «обращение→результат за N минут» — только Digital-Pro (сильно) и частично ROBIN. В вебе — n8n/Gumloop канвы.
  • Metrics-first кейсы — ROBIN/NIKTA/Generation AI; у большинства RU (Webseed, Code9, Automacon) кейсы без цифр.
  • Именованная команда/инженер — Digital-Pro (1 фото), Insight (27 специалистов), NIKTA (CEO). У остальных — аноним.

4. Язык клиента (дословно — использовать в копирайте)

См. 03-reviews/pains-verbatim.md. Ключевые: «полчаса пишу одно и то же», «пять вкладок и Excel», «суббота 21:47 → понедельник 10:12», «2–4 часа в день на типовые вопросы», «3–8 млн ₽ в год теряет компания из 100 человек», «нет множества необходимых функций» (о конструкторах).

5. Визуальные конвенции (must-have, чтобы выглядеть в нише достоверно)

Из раунда 1: pipeline-схема процесса, полоса интеграций с ИМЕНАМИ систем (1С, amoCRM, Bitrix24, МойСклад — не безликие иконки), крупные метрики, таймлайн этапов. Сайт уже имеет: оранжевая палитра #F97316/#FFFBF5, Inter, timeline+код-анимация в sb-plan.

6. Паттерны доверия лидеров

RU: логотипы клиентов + цифры кейсов (ROBIN, NIKTA, Generation AI), именованный инженер (Digital-Pro), награды (OSMI). Запад: живое демо (Bland/Vapi — сильнейший сигнал «реально работает»), compliance-стена (Sierra).

7. Паттерны конверсии лидеров

  • CTA лидеров: «Запросить пилот» (Just AI), «Начать аудит» (scriptogenerator), «Обсудить проект».
  • Сильнейшая конверсионная механика ниши: диагностика с ощутимым артефактом на выходе (ТЗ/план/ROI) вместо «оставьте заявку».
  • Вывод для BetaLine: CTA «Опишите затык — получите план» с реальным разбором в боте, эскалация на живого инженера через существующий api/chat.js-паттерн.

Limitations

  • Firecrawl без кредитов → транспорт WebSearch/WebFetch; глубина скрейпа отзывов ограничена (otzovik 507, startpack 403, astanahub paywall). Боли подтверждены 4 независимыми источниками, но выборка по конструкторам ~15 цитат.
  • B2B-ниша: отзывы маркетплейсов неприменимы; опора на отраслевые блоги/обзорники (usedesk, sostav, vc.ru, crmindex) — это вторичные носители языка клиента, первичные форум-треды не скрейпились.
  • Phase 4 (8-критериальный скоринг + HTML-дашборд) свёрнута в раунд-1 отчёты и этот gap-анализ по решению: клиентский артефакт — обновляемый artifact-proposal, отдельный дашборд не нужен пользователю.
